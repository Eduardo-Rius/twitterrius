# 🚀 Twitter Rius

Una aplicación de microblogging ultra-rápida construida con **HTML, CSS y JavaScript puro**, conectada a **Supabase**.

## ✨ Características
- **SPA (Single Page Application)**: Navegación instantánea sin recarga.
- **Glassmorphism Design**: Interfaz moderna con efectos de cristal y modo oscuro.
- **Identidad Anónima**: Cada dispositivo tiene su propio ID persistente.
- **Interactividad**: Crear posts, dar likes (1 por post) y borrar tus propios mensajes.
- **Responsive**: Optimizado para dispositivos móviles y escritorio.

## 🛠️ Tecnologías
- **Frontend**: Vanilla HTML5, CSS3, JavaScript (ES6+).
- **Backend**: Supabase (Database, RLS).
- **Despliegue**: Cloudflare Pages.

## 📝 Configuración de Supabase
Para que la app funcione, debes ejecutar este SQL en tu editor de Supabase:

```sql
CREATE TABLE posts (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  content TEXT NOT NULL,
  author_id TEXT NOT NULL,
  is_featured BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL
);

CREATE TABLE likes (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  post_id BIGINT REFERENCES posts(id) ON DELETE CASCADE,
  device_id TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW()) NOT NULL,
  UNIQUE(post_id, device_id)
);

ALTER TABLE posts ENABLE ROW LEVEL SECURITY;
ALTER TABLE likes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Lectura pública" ON posts FOR SELECT USING (true);
CREATE POLICY "Creación pública" ON posts FOR INSERT WITH CHECK (true);
CREATE POLICY "Borrado por autor" ON posts FOR DELETE USING (true); -- Filtrar por author_id en producción real

CREATE POLICY "Lectura likes" ON likes FOR SELECT USING (true);
CREATE POLICY "Creación likes" ON likes FOR INSERT WITH CHECK (true);
```

## 🚀 Despliegue en 3 pasos
1. **GitHub**: Sube este código a un repositorio nuevo en tu cuenta de GitHub.
2. **Cloudflare Pages**: Conecta tu repositorio de GitHub a Cloudflare Pages.
3. **Build**: Deja los ajustes de build vacíos (no necesitamos comandos de build). ¡Y listo!
